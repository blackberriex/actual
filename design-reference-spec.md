# Design Reference Spec — Vault Money

Детальний розбір референсів (HR-дашборд, зобр. 1–3) і план переробки
поточного UI проєкту (Actual/Vault, зобр. 4–5) під цей стиль.

Документ доповнює палітри [monochrome-dark-palette.md](monochrome-dark-palette.md)
і [monochrome-light-palette.md](monochrome-light-palette.md): тут — форма,
відступи, типографіка, компоненти й конкретні зміни по сторінках.

## ЧАСТИНА A. Розбір референсу (що саме копіюємо)

### A1. Загальне враження

- **Глибокий нейтральний чорний фон** (`#0e0e0f`–`#161617`), на ньому
  «плавають» картки трохи світліші (`#1b1b1d`) з **великим радіусом
  заокруглення** (20–24px) і ледь помітним бордюром.
- **Монохром + один кислотний акцент** — лаймово-зелений `#c8f23f`
  (chartreuse). Акцент трапляється рідко: активний бар графіка, лого,
  головна CTA-кнопка, тогл. Усе інше — відтінки сірого та білий.
- **Дуже великі, жирні цифри** як головний герой кожної картки
  (220, 98%, 05). Поряд — дрібний uppercase-лейбл і кольоровий
  дельта-бейдж (↑2.5% / ↓4.1%).
- Багато **повітря** (padding 24–32px у картках), мало ліній-розділювачів —
  межі задаються фоном і відступами, а не бордерами.

### A2. Сітка та відступи (spacing scale)

| Рівень                       | Значення               | Де                        |
| ---------------------------- | ---------------------- | ------------------------- |
| Базова одиниця               | 4px                    | весь ритм кратний 4       |
| Внутрішній padding картки    | 24–28px                | усі картки/панелі         |
| Gap між картками             | 16–20px                | сітка дашборду            |
| Зовнішній padding контенту   | 28–32px                | від краю в'юпорту         |
| Радіус картки                | 20–24px                | панелі, модалки           |
| Радіус кнопки/інпута         | 12–14px                | контроли                  |
| Радіус pill/badge            | 8px або повний (999px) | теги, статуси             |
| Радіус іконкового контейнера | повний круг            | круглі іконки в KPI       |
| Висота рядка таблиці         | ~64px                  | просторі рядки з аватаром |
| Висота кнопки/інпута         | 40–44px                | контроли                  |

**Головний висновок для нашого проєкту:** зараз усе занадто щільне і
гострокутне. Потрібно збільшити радіуси (4–6px → 12–20px) і padding.

### A3. Типографіка

- Гротеск (схоже на **Inter / Geist / General Sans**). Для нас — **Inter**.
- Шкала:
  - KPI-число: **48–64px, weight 700**, tight letter-spacing (-0.02em).
  - Заголовок секції («Overview», «Attendance Overview»): **18–20px, 600**.
  - Лейбли над KPI: **11–12px, 600, uppercase, letter-spacing 0.06em**,
    колір subdued.
  - Body / таблиця: **14px, 400–500**.
  - Дрібні підписи (ID, дати): **12px, 400**, колір subdued.
- **Tabular numbers** обов'язково на всіх числах (`font-variant-numeric:
tabular-nums`).
- Дельта-бейджі: число + стрілка в кольоровому pill з напівпрозорим фоном
  (`rgba(green,0.12)` / `rgba(red,0.12)`).

### A4. Іконки

- **Лінійні (outline) іконки, stroke ~1.5px**, розмір 20px, у наборі
  єдиного стилю (схоже на Iconoir / Lucide). Для нас — **Lucide**.
- У сайдбарі — іконка 20px + лейбл 14px, gap 12px.
- У KPI-картках іконка лежить у **круглому контейнері** (44×44px,
  фон `#222`, іконка біла/сіра).
- Жодних емодзі. Жодних filled-іконок упереміш з outline.

### A5. Сайдбар (детально)

- Ширина ~240px, фон трохи темніший/рівний фону сторінки.
- Зверху — **лого** (монограма + лаймовий кружок).
- Пункти меню: іконка + текст, padding по вертикалі 10–12px, радіус 12px.
- **Активний пункт = інверсія / світлий блок:** у референсі активний
  «Dashboard» — це світлий темно-сірий блок (`#1f1f22`) з білим текстом і
  тонким бордюром; решта пунктів — лише сірий текст без фону.
  - У **нашій** монохромній системі йдемо ще далі: активний пункт = **білий
    блок + чорний текст** (повна інверсія, як ми домовились у палітрі).
- Hover неактивного пункту: ледь помітний фон (`#1c1c1e`), текст світлішає.
- Пункт із підменю (Projects) має шеврон-стрілку праворуч.
- **Внизу сайдбару** — картка користувача: аватар + ім'я + email, поряд
  «...» меню, і **сегментований перемикач Light / Dark** (pill зі слайдером).

### A6. Картки KPI (верхній ряд)

Структура зверху вниз:

1. Кругла іконка (44px).
2. Uppercase-лейбл + «?»-підказка (tooltip).
3. Велике число.
4. Дельта-pill (↑/↓ %) + сірий текст «vs last month».
5. Праворуч від числа — **міні bar-chart** (sparkline) у колір дельти
   (зелений, якщо ріст; червоний/кораловий, якщо спад).

### A7. Графіки

- **Heatmap (Attendance):** сітка клітинок із заокругленням ~6px, gap ~6px,
  градація від `#1c1c1e` (нуль) до чисто білого (пік) — монохромна теплова
  карта. Підписи осей — 12px subdued.
- **Bar-chart (Project Overview):** стовпці-«пігулки» з великим радіусом
  (повне заокруглення зверху), сірі, а **активний/виділений стовпець —
  лаймовий**. Над ним — tooltip білою пігулкою з днем і значенням, і точка-
  маркер лаймова.
- **Donut (Department):** товсте кільце, сегменти різнокольорові (тут
  допускається кольоровість бо це категорійні дані), у центрі порожньо,
  hover-сегмент піднімає tooltip. Легенда — кольорові крапки + лейбл.
- Загальний принцип графіків: **сітка/осі майже невидимі**, дані — головні.

### A8. Таблиця (Attendance / All Employees)

- Header-рядок: 12px uppercase subdued, без фону або ледь темніший.
- Рядок: аватар (круглий 36px) + ім'я (14px 600) + під ним ID (12px subdued).
- Статуси — **pill-бейджі**: «Attend» зелений outline-pill, «Day off»
  червоний, «Full-time» зелений, «Contract» фіолетовий тощо.
- Чекбокс зліва, «...» дій справа.
- Рядки високі (~64px), розділювач — дуже тонка лінія або просто відступ.
- Hover рядка — ледь світліший фон.

### A9. Картки-розклад (Schedule, права колонка)

- Вкладки-сегмент зверху: All / Meetings / Events (активна — світлий pill).
- Кожна подія — окрема картка з бордюром, радіус 16px:
  - Назва (14px 600), дата + час (12px subdued).
  - Внизу — іконка сервісу (Google Meet / Zoom) + назва + стрілка, праворуч —
    стек аватарів учасників (overlapping).

### A10. Кнопки

- **Primary CTA** («Add New Employee») — лаймова заливка, чорний текст,
  радіус 12px, padding 12×20px, weight 600.
- **Secondary** («This week ⌄», «Filter») — прозорий/темний фон, тонкий
  бордюр, сірий текст, радіус 12px.
- **Дата-пікер** у хедері — pill із іконкою календаря + текст дати + бордюр.

### A11. Модалки (екстраполяція стилю)

У референсі модалок немає, але за системою вони мають бути:

- Фон `#1b1b1d`, радіус 20–24px, padding 28px, бордюр `#2a2a2c`.
- Оверлей `rgba(0,0,0,0.5)`.
- Заголовок 18px 600 + кнопка-хрестик (bare icon) у куті.
- Кнопки внизу праворуч: Primary (білий/лаймовий) + Secondary (bare).

## ЧАСТИНА B. Що зробити в нашому проєкті (по сторінках)

> Поточний стан (зобр. 4–5): функціонально все є, але візуально щільно,
> гострокутно, мало ієрархії, числа дрібні, забагато тонких ліній.

### B1. Глобально (стосується всіх сторінок)

1. **Перейти на монохромну палітру** з палітр-файлів (анкер `#161617`).
   Прибрати синій підтон сірих.
2. **Збільшити радіуси:** картки/панелі 6px → **16–20px**, кнопки/інпути →
   **12px**, бейджі → pill.
3. **Збільшити padding** усередині панелей до 24px, gap між блоками до 16px.
4. **Інверсія активного елемента** в сайдбарі (білий блок + темний текст) —
   замість поточного сірого виділення.
5. **Замінити іконки** на єдиний outline-набір (Lucide), stroke 1.5px, 20px.
6. **Tabular numbers** скрізь де гроші (вже є `FinancialText`/`styles.tnum` —
   застосувати послідовно).
7. **Один акцент.** Обрати: або лишити білий (як у палітрі), або додати
   лаймовий `#c8f23f` як єдиний акцент (CTA, активний бар графіка). Не
   змішувати кілька яскравих кольорів — лишити green/red лише для сум.
8. **Прибрати зайві бордюри** — глибину давати фоном (елевація), а не
   лініями.

### B2. Сторінка «Бюджет» (зобр. 4)

- **Картки місяців** («червня», «липня») зробити **кругліше (радіус 20px)**
  і з більшим внутрішнім padding (зараз тісно). Додати ледь помітний
  бордюр `#2c2c30` замість різкого краю.
- **«Перевищення бюджету: -829.87»** — це головна цифра картки. Зробити її
  **великою (32–40px, 700)**, як KPI в референсі, а лейбл над нею — дрібним
  uppercase subdued.
- Рядки 0.00 у таблиці зараз сірі й зливаються — це ок (нульові значення
  мають бути приглушені `budgetNumberZero`), але **збільшити висоту рядка**
  (зараз тісно) і **відступ між групами** категорій.
- **Заголовки груп** («Постійні витрати», «Щоденні витрати») зробити
  жирнішими (600) і додати трохи більше вертикального простору навколо.
- Колонкові заголовки (Призначено/Витрачено/Баланс) — 12px uppercase subdued.
- Hover рядка категорії — підсвітка `#1c1c1e`.
- Заокруглити кути всієї таблиці-контейнера (радіус 16px, overflow hidden).

### B3. Сторінка «Звіти» (зобр. 5)

- **KPI-картки** (Total Income, Total Expenses, Avg Per Month, Avg Per
  Transaction) — це найближче до референсу. Допрацювати:
  - Збільшити число до **36–44px 700**, лейбл — uppercase 12px subdued.
  - Період («січ. 2026 – черв. 2026») — 12px subdued під лейблом.
  - Радіус картки 20px, padding 24px, gap між картками 16px.
  - Додати дельта-pill (↑/↓ vs попередній період), якщо є дані.
- **Графік «Чистий капітал»** — лишити зелену area-лінію, але прибрати зайві
  осі; додати тонку сітку лише за потреби. Це наш «акцентний» графік.
- **This Month / Budget Overview / 3-Month Average** — вирівняти радіуси й
  padding з KPI-картками; пунктирні лінії лишити сірими.
- Розкласти все на єдину сітку 16px gap, однакові радіуси 20px.

### B4. Сайдбар (зобр. 4–5)

- **Логотип «Vault»** зверху — додати монограму/значок (як «K●» у референсі),
  можна з лаймовою крапкою-акцентом.
- Пункти меню (Бюджет, Звіти, Розклади, Більше) — іконка Lucide 20px + текст
  14px, padding 10–12px, радіус 12px, gap 12px.
- **Активний пункт** («Бюджет») — зараз сірий блок; зробити **білий блок +
  темний текст** (інверсія).
- Підменю «Більше» (Отримувачі, Правила, Теги, Налаштування) — з відступом
  зліва, дрібніше, subdued, з тонкою вертикальною лінією-гідом за бажанням.
- **«Всі рахунки 2,067.43»** — лейбл uppercase subdued + число tabular,
  вирівняне праворуч. «В межах бюджету», «Privat UAH» — той самий патерн.
- **«+ Додати рахунок»** внизу — bare-кнопка з іконкою plus, hover-фон.
- Додати **картку користувача внизу** з перемикачем Light/Dark (як у
  референсі) — сегментований pill.

### B5. Хедер сторінки

- Як у референсі «Employee»: зліва назва сторінки (24px 700), справа —
  дата-pill (іконка календаря + дата + бордюр) і Primary CTA (наприклад
  «Додати транзакцію») лаймовим/білим.
- Бейдж «29 транзакцій без категорії» зробити **warning-pill** (фон
  `rgba(amber,0.12)`, текст amber, радіус pill) замість простого червоного
  тексту.
- Іконки в хедері (місяць, око, довідка) — outline 20px, у bare-кнопках із
  hover-фоном.

### B6. Таблиця транзакцій

- **Збільшити відступи / висоту рядка** (зараз дуже щільно) до ~52–64px.
- Категорія/payee — 14px 500; сума — tabular, вирівняна праворуч,
  колір за знаком (`numberPositive`/`numberNegative`).
- Статуси (cleared/uncleared, перекази) — **pill-бейджі** в стилі референсу.
- Hover рядка — `#1c1c1e`; обраний рядок — тонкий білий бордюр зліва або
  легка підсвітка.
- Header — 12px uppercase subdued, sticky.
- Заокруглити контейнер таблиці (16px) і прибрати зайві вертикальні лінії
  між колонками.

### B7. Модалки (Settings, додавання рахунку/категорії, тощо)

- Радіус 20px, padding 28px, фон `modalBackground`, бордюр `modalBorder`,
  оверлей `rgba(0,0,0,0.5)`.
- Заголовок 18px 600 + хрестик-іконка (bare) у правому куті.
- Інпути: радіус 12px, висота 40–44px, фон `formInputBackground`, бордюр
  `formInputBorder`, focus — білий бордюр + легка тінь.
- Лейбли полів — 12px 600 subdued над інпутом.
- Кнопки внизу праворуч: Primary (білий/лаймовий, чорний текст) +
  Secondary/Cancel (bare).
- Перемикачі (toggle) — у стилі сегментованого pill зі слайдером.

### B8. Налаштування

- Згрупувати в **картки-секції** (радіус 16px, padding 24px, заголовок
  секції 16px 600 + опис 13px subdued).
- Кожен рядок налаштування: лейбл + опис зліва, контрол (toggle / select /
  кнопка) справа, вертикальний padding 14–16px, тонкий розділювач між
  рядками `tableBorderSeparator`.
- Toggle-перемикачі — `checkboxToggle*` кольори з палітри; активний — білий.
- Select/dropdown — стиль secondary-кнопки (бордюр + шеврон).

## ЧАСТИНА D. Зафіксоване рішення — акцент

**Обрано: D2 — монохром + лайм.**

База монохромна, єдиний кольоровий акцент — лаймовий `#c8f23f` (текст на
ньому `#161617`). Додано в обидві палітри:

```css
--color-accent: #c8f23f;
--color-accentHover: #b8e22f;
--color-accentText: #161617;
```

**Правила застосування акценту:**

- **Лайм — рідкісний і головний.** Одне-два використання на в'ю: primary
  CTA, активний бар графіка, лого-крапка, ключове виділення. Не фарбувати
  лаймом усі активні стани.
- **Білий (dark) / чорний (light) — нейтральна інверсія** «це обрано»:
  активний пункт сайдбару, selected row/pill. Це не акцент, а інверсія.
- **Dark:** `buttonPrimary*` → лаймовий (вже оновлено в палітрі).
- **Light:** `buttonPrimary*` залишено **чорним** заради контрасту тексту;
  лайм там — для активного бару графіка, лого й виділень. За бажанням можна
  зробити CTA лаймовим і на світлій темі, але тоді текст лишати `#161617` і
  тестувати контраст на дрібних кнопках.
- **Green/Red** (`numberPositive`/`numberNegative`) лишаються виключно для
  сум — не плутати з акцентом.
